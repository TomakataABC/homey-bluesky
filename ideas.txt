- [x] Delete own post

- Add and remove users from lists

- Change notification preferences

Important technical notes

- Most read and event features will likely require polling.
- Bluesky has a firehose for public network updates, but it is not the simplest source for per-account Homey automations.
- Notifications, mutes, some preferences, and chat are authenticated or private.
- Media support adds extra complexity.
- Moderation labels and visibility rules must be respected in widgets and data displays.

Suggested roadmap

Phase 1
- Create post
- Reply to post
- Like and repost
- New mention trigger
- New follower trigger
- Unread notification count
- Profile summary widget

Phase 2
- Post with images
- Follow and unfollow
- Watched author posted trigger
- Post engagement triggers
- Latest post widget
- Notifications widget

Phase 3
- List support
- Profile update actions
- Moderation actions
- Custom feed monitoring
- Thread tracking
- DM features if they prove useful

Summary

Bluesky is capable of supporting a much broader Homey integration than posting alone. The best areas for expansion are notifications, engagement tracking, profile and graph status, feed monitoring, and a small set of focused widgets.
